# Frontend

Angular 21 SPA with standalone components, Signals, and Angular Material.

## Project Structure

```
frontend/
├── src/
│   ├── app/
│   │   ├── core/
│   │   │   ├── models/          # TypeScript interfaces (Pack, Question, Script, Settings)
│   │   │   ├── services/        # AuthService, BedrockService, StorageService, etc.
│   │   │   ├── guards/          # Route guards (auth, login)
│   │   │   └── utils/           # Prompt builders, domain inference, file splitter
│   │   ├── features/
│   │   │   ├── login/           # Cognito login page
│   │   │   ├── question-input/  # Paste exam question + generate review
│   │   │   ├── question-list/   # List of reviewed questions
│   │   │   ├── review-viewer/   # Markdown viewer + edit + refine with AI
│   │   │   ├── transcripts/     # Transcript input + script viewer
│   │   │   ├── chat/            # Open chat (tutor conversation + summary)
│   │   │   ├── packs/           # Pack editor + drawer
│   │   │   ├── export/          # Export to Markdown / ZIP
│   │   │   ├── settings/        # User preferences
│   │   │   └── methods/         # Study method selector
│   │   └── shared/
│   │       ├── components/      # DomainBadge, ThemeToggle, ConfirmDeleteDialog, etc.
│   │       └── pipes/           # TruncatePipe
│   ├── environments/            # environment.ts (generated by Terraform)
│   └── styles/                  # SCSS variables and reset
├── public/
│   ├── examples/                # Pack template JSONs (canonical source)
│   └── logo/                    # App logo assets
└── package.json
```

## Key Services

| Service | Purpose |
|---------|---------|
| `AuthService` | Cognito SRP login, token management, session restore |
| `BedrockService` | Streams reviews/refinements via `POST /converse` |
| `StorageService` | DynamoDB sync (initialize, CRUD, migration from localStorage) |
| `ModelsService` | Fetches available models from `GET /data/models` with static fallback |
| `QuestionsService` | Local state for questions (CRUD, selection, domain breakdown) |
| `PacksService` | Pack CRUD, active pack management |
| `ChatService` | Chat session CRUD, message append/streaming, summary management |
| `ExportService` | Build Markdown content, download files, ZIP for multi-file exports |

## Review Prompt Engineering

The system prompt is built dynamically in [`core/utils/review-prompt.util.ts`](../frontend/src/app/core/utils/review-prompt.util.ts). Key features:

- **Input parsing instructions:** 5-step structured approach teaching the model to identify question text, options (in original order), and correct answers from raw pasted practice exam text
- **Status marker handling:** explains platform artifacts like "Sua resposta está incorreta", "Resposta correta"
- **Output format:** fixed Markdown template with Key Concepts, Question, Alternatives, Correct answer, Incorrect answers
- **Language support:** translations in "Question" and "Alternatives" sections; explanations only in the selected language
- **Ordering rules:** alternatives in original order, correct/incorrect in ascending letter order
- **Structural example:** a generic sample showing the expected output format

### Pack JSON structure

```json
{
  "name": "Certification full name",
  "description": "Short description (used in AI prompt for classification)",
  "version": "Optional label (e.g., practice exam source)",
  "color": "#hex",
  "domains": [
    { "name": "Domain Name", "description": "...", "order": 1 }
  ],
  "exportIntroQuestions": "Markdown intro for exported question files",
  "exportIntroTranscripts": "Markdown intro for exported transcript files",
  "exportIntroChat": "Markdown intro for exported chat files"
}
```

## Export System

- **Single file:** downloads a `.md` with title + `exportIntroQuestions` + questions grouped by domain
- **Split mode:** toggle "Split into multiple files" → generates a `.zip` containing balanced `.md` parts (each with full title + intro)
- **By domain:** download all questions from selected domains
- **File naming:** `{pack-name}-{suffix}.md` or `{pack-name}-N-parts.zip`
